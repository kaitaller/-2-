## 🧩 Описание классов

### Пространство имён `Color`

Хранит ANSI-коды для цветного вывода в консоли.

| Поле      | Тип         | Значение                        |
|-----------|-------------|----------------------------------|
| `RESET`   | `std::string` | Сброс цвета (`\033[0m`)         |
| `RED`     | `std::string` | Красный (`\033[31m`)            |
| `GREEN`   | `std::string` | Зелёный (`\033[32m`)            |
| `YELLOW`  | `std::string` | Жёлтый (`\033[33m`)             |
| `CYAN`    | `std::string` | Голубой (`\033[36m`)            |
| `BOLD`    | `std::string` | Жирный (`\033[1m`)              |
| `DIM`     | `std::string` | Тусклый (`\033[2m`)             |

> Используются только в `std::cout` для оформления.

### Перечисление `PuzzleType`

Определяет тип головоломки.

| Значение    | Описание                |
|-------------|-------------------------|
| `NONE`      | Нет головоломки         |
| `RIDDLE`    | Словесная загадка       |
| `MATH`      | Математическое уравнение|
| `SEQUENCE`  | Продолжить числовой ряд |

---

### Структура `Puzzle`

Хранит данные об одной головоломке и логику её решения.

#### Поля

| Поле       | Тип            | Описание                                    |
|------------|----------------|---------------------------------------------|
| `type`     | `PuzzleType`   | Тип головоломки                             |
| `question` | `std::string`  | Текст вопроса                               |
| `answer`   | `std::string`  | Правильный ответ (в нижнем регистре)        |
| `hint`     | `std::string`  | Подсказка для игрока                        |

#### Методы

##### `bool solve() const`

Запускает процесс решения головоломки.

- **Входные данные**: отсутствуют (используются поля структуры)
- **Выходные данные**:
  - `true` — игрок дал правильный ответ за ≤3 попыток
  - `false` — ответ не угадан
- **Побочные эффекты**: вывод в консоль вопроса, подсказки, результатов попыток

> 🔁 Игроку даётся 3 попытки. Ответ приводится к нижнему регистру перед сравнением.

---

### Структура `Location`

Описывает одну локацию (комнату) лабиринта.

#### Поля

| Поле           | Тип                     | Описание                                      |
|----------------|-------------------------|-----------------------------------------------|
| `id`           | `int`                   | Уникальный номер локации                      |
| `name`         | `std::string`           | Название (например, "Зал Загадок")            |
| `map`          | `std::vector<std::string>` | ASCII-карта локации (массив строк)         |
| `connections`  | `std::vector<int>`      | Список ID локаций, куда можно перейти         |
| `isOpen`       | `bool`                  | Доступна ли локация для входа                 |
| `isExit`       | `bool`                  | Является ли локация выходом (концом игры)     |
| `puzzle`       | `Puzzle`                | Головоломка, которая охраняет вход (если есть)|

> Методы у структуры отсутствуют.

### Структура `Character`

Представляет игрока.

| Поле         | Тип   | Описание                       |
|--------------|-------|--------------------------------|
| `locationId` | `int` | ID текущей локации игрока      |

> Методов нет.

---

### Класс `Game`

Управляет игровым процессом: инициализацией, ходом игры, перемещением, проверкой условий победы.

#### Поля

| Поле        | Тип                         | Описание                          |
|-------------|-----------------------------|-----------------------------------|
| `locations` | `std::map<int, Location>`   | Словарь всех локаций по ID        |
| `player`    | `Character`                 | Текущее состояние игрока          |

#### Методы

##### `Game()`

Конструктор. Вызывает `init()` для начальной настройки игры.

---

##### `void run()`

Главный игровой цикл.

- **Входные данные**: нет
- **Выходные данные**: нет (завершает программу при выходе или победе)
- **Логика**:
  1. Отображение приветствия
  2. Бесконечный цикл:
     - Отображение текущей локации
     - Проверка победы (`isGameOver()`)
     - Показ доступных переходов
     - Ввод команды от игрока
     - Обработка перемещения или выхода

---

##### `void init()`

Инициализирует лабиринт: создаёт 4 предопределённые локации, устанавливает связи, головоломки.

- **Локация 1**: Тёмный вход (открыт, стартовая) → головоломки нет
- **Локация 2**: Зал Загадок → загадка про время
- **Локация 3**: Комната Счисления → уравнение `3*X+7=22`
- **Локация 4**: Врата Свободы → последовательность Фибоначчи

---

##### `void clearScreen()`

Очищает консоль кроссплатформенно (`cls` на Windows, `clear` на Linux/macOS).

---

##### `void printTitle()`

Выводит ASCII-арт логотипа игры и подзаголовок.

---

##### `void printLocation(const Location& loc)`

Выводит карту локации и легенду символов.

- **Входные данные**: ссылка на объект `Location`
- **Выходные данные**: нет (вывод в консоль)

---

##### `void printConnections()`

Показывает все доступные переходы из текущей локации с указанием статуса (открыта/закрыта).

- Использует `player.locationId` для получения текущей локации

---

##### `void handleMove(int targetId)`

Обрабатывает попытку игрока перейти в другую локацию.

- **Входные данные**: `targetId` — ID целевой локации
- **Логика**:
  1. Проверяет, есть ли переход
  2. Если целевая локация закрыта — запускает `puzzle.solve()`
  3. При успешном решении открывает локацию
  4. Перемещает игрока

- **Выходные данные**: нет (меняет состояние игры)

---

##### `bool isGameOver() const`

Проверяет, достиг ли игрок выхода.

- **Входные данные**: нет
- **Выходные данные**: `true`, если текущая локация имеет `isExit == true` и `isOpen == true`

## 🚀 Инструкция по запуску

### Требования

- Компилятор C++11 или новее (g++, clang++, MSVC)
- Терминал с поддержкой ANSI-цветов (в Windows 10+ — работает, в более старых — возможно, потребуется включить виртуальный терминал)

### Компиляция и запуск

#### Linux / macOS (терминал)

```bash
g++ -std=c++11 maze_quest.cpp -o maze_quest
./maze_quest
